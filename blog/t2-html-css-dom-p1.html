Can you think of an analogy to describe HTML, CSS and the DOM? How would you describe it to your non-technical friend?

A good way to explain the HTML, CSS and the DOM and the relevance of them to each other would be much like building a house, the materials and how to read the blueprints in an easy to read format.

HTML

HTML is like the frame work of the house. HTML will tell you...

  • How many levels the house has
  • How many rooms are on each level
  • Where each room is positioned
  • How each room positioned in comparison to other rooms
  • The contents within each room/level like does it have carpet, windows, lights etc

CSS

CSS is the material, and wallpaper, carpet and colours of each for each room, level, wall, window etc - It's a set of materials that helps the HTML know what it should be using to create each of the different elements within the overall house design.

DOM

DOM is a way of viewing all of the HTML and CSS in a manner similar to a blueprint. It helps display the content of HTML and the CSS in a way which is easy to manage and read. If you were to give your "blueprint" to someone who knows how to read it, they will easily be able to structure and display your HTML and CSS in the way you intended it to be by breaking it up into the levels, rooms, materials and where each material should be used etc.

What is meant by boxifying design?

Boxifying design is to basically image a website broken up into boxes. So imagine havine a grid layered over a website design, from there you can see what elements will fit into boxes. Some boxes will contain boxes within them and that's to help section off certain areas with "div" commands so it becomes easier to know the heirarchy of the elements and what their parent container/box is.

What is the box model?

The box model is to give each element substance to a degree. It all starts with content, which then has a margin, borders, padding etc. This allows the developer to shape/position the content in a way that makes that content sit in a clear position against the surrounding content.
